# Current Progress (State)

學生已經穩固了基礎文法核心與進階句構。目前已掌握以下【多益文法核心架構 V3.6】：

## 模組一：詞性基礎與核心概念 (Parts of Speech & Core Concepts)
* 代名詞規則：所有格有專屬單字（hers, yours, its），嚴禁畫蛇添足加上 's。
* 冠詞發音判定：a / an 完全取決於單字開頭的「發音」，而非拼字。陷阱題：an hour, a university。
* 定冠詞與零冠詞系統 (Articles System)：the 為絕對記憶體指標 (Absolute Pointer)，用於特指、唯一或最高級；零冠詞代表全域抽象變數 (Global Abstract Variable) 或不可數/複數的泛指概念。
* 不可數名詞限制：抽象概念或整體總稱（furniture, information, advice）永遠視為單數，嚴禁加 s 或 a/an。

## 模組二：介系詞與邏輯連接詞 (Prepositions & Logical Connectors)
* 空間與時間概念 (In/On/At)：
    * In (大範圍/區間)：in 2026, in Taiwan.
    * On (特定接觸面/特定日子)：on Monday, on the desk.
    * At (精確點/時刻)：at 10:30, at the station.
* 邏輯連接詞互斥原則：Because 與 So 絕對互斥（不可同句）；Although 與 But 絕對互斥。
* 副詞子句連接協定 (Adverbial Connectors)：精準定義時間 (when/while)、原因 (because/since)、條件 (if/unless) 與讓步 (although/even if) 的觸發條件。

## 模組三：動詞系統與時態規則 (Verb System & Tense Rules)
* 動詞層級分類：
    * Be 動詞：表達狀態 (=)，用於形成被動與進行式。
    * 助動詞：do/does/did/will/can，負責形成否定與疑問句。
    * 一般動詞：表達具體動作。
* 時態三大鐵律與 12 時態核心：
    * 鐵律一：句子中永遠只有「最前面」的第一個動詞負責表現時間戳記（現在/過去）。
    * 鐵律二：遇到助動詞 (do/does/did/will/can) 後，主動詞強制「還原為原形」。
    * 鐵律三：遇到完成式助動詞 (have/has/had) 後，主動詞強制變為「過去分詞 (V.p.p.)」。
    * 完成式快取 (Perfect Tense Cache)：現在/過去/未來完成式作為狀態的歷史日誌 (Historical Log)，記錄截至某時間點的累積數據。
* 情態動詞權重 (Modal Logic)：must / may / can / should 代表執行機率與推測權重，或用於表達強制的系統權限 (Permissions)。
* 使役動詞用法 (Causative Verbs)：
    * 主動委託：have + [人] + [原形動詞] (請某人做某事)。
    * 被動狀態：have + [物] + [V.p.p.] (讓某事被完成)。

## 模組三點五：非限定動詞（動詞封裝協定 - Verb Wrapper Protocol）
* 型別轉換邏輯：當動詞不擔任主執行點時，須封裝為名詞、形容詞或副詞屬性。
* 不定詞 (to + V)：非同步任務 (Async Task)，代表「尚未發生」、「目的」或「未來導向」。
* 動名詞 (V-ing)：狀態快照 (Snapshot)，代表「經驗」、「事實」或「已存在之程序」。
* 動態記憶體分配 (Dynamic Allocation)：stop / remember / forget 依據後接 to-V (尚未執行的 Task) 或 V-ing (已執行的 Memory) 改變整體句意。
* 封裝衝突判定：由主動詞（Admin）權限決定。意圖型（decide, plan）接 to-V；經驗/過程型（enjoy, avoid）接 V-ing。

## 模組四：子句結構與修飾語 (Clause Structures & Modifiers)
* 三大子句架構 (Clause Architecture)：正式成型為名詞子句 (作主/受詞)、形容詞子句 (修飾名詞) 與副詞子句 (修飾全句)。
* 名詞子句 (Noun Clauses)：包含 that / whether/if / wh- 結構，語序還原為主詞+動詞。
* 關係子句 (形容詞子句)：who, which, that, whose。若關代在子句中作受詞則可省略。
* 關係副詞與非限定用法：引入 where / when / why 作為環境變數修飾；使用逗號「, which/who」進行非限定補充說明 (Non-restrictive)。

## 模組五：分詞構句 (Participle Phrases - 句型壓縮協議)
* 壓縮先決條件：主要子句與附屬子句的主詞必須「邏輯一致」。
* 標準壓縮 SOP：
    1. 刪除連接詞 (可保留以強調時間)。
    2. 刪除重複主詞。
    3. 動詞轉換：主動轉 V-ing / 被動轉 V.p.p.（省略 being）。
* 關係子句縮減 (Reduced Relatives)：形容詞子句的直達壓縮路徑，直接刪除關代與 Be 動詞，留下 V-ing (主動) 或 p.p. (被動) 作為後位修飾。
* 邊界條件 (獨立分詞構句)：若主詞不同，必須保留附屬子句主詞作為「區域變數」，僅對動詞進行主被動轉換。
* 核心禁忌：主要子句（主程式）的時態與結構神聖不可侵犯，嚴禁改動。

## 模組六：假設語氣與條件句 (Subjunctive Mood & Conditionals - 虛擬機環境系統)
* 條件句層級 (Conditionals 0/1/2/3)：從絕對事實 (Level 0) 到與過去事實相反 (Level 3)，以及跨越時間軸的混合假設 (Mixed Conditionals)。
* 核心偏移公式 (Timeline Offset)：$Reality - 1 = Subjunctive$。透過時態往過去偏移，建立與現實的邏輯距離。
* 環境狀態定義：
    * 現在虛擬 (Present Unreal)：對現狀腦補，時態 -1（過去式），Be 動詞初始化為 were。
    * 過去虛擬 (Past Unreal)：對遺憾補丁，時態 -2（過去完成式 had V.p.p.）。
    * 機率觸發器 (Should)：極低機率 (1%~5%) 的異常處理程序 (Exception Handler)，語意為「萬一」。
* 倒裝協定 (Inversion)：卸載 If 標頭，將 Had / Were / Should 提至句首擔任入口點。
* 強制還原協定 (Mandatory Subjunctive)：Admin 指令（suggest, insist, essential）觸發，that 子句動詞強制回歸「原型 (Base Verb)」。

## 模組六點五：進階句型重構 (Advanced Syntax Refactoring)
* 否定倒裝 (Negative Inversion)：當否定副詞提至句首時，強制觸發「助動詞提升 (Auxiliary Hoisting)」至主詞前方 (例：Never have I...)。
* 強調句型 (Cleft Sentences)：使用 $It \ is/was \ ... \ that \ ...$ 框架來高光標記特定變數 (主詞、受詞或時間地點)。
