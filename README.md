## 模組一：基礎變數宣告與資料型態 (Variables & Data Types)
1. 代名詞最高權限：內建變數（你我他）所有格自帶專屬格式，嚴禁附加 's (例如：hers, yours, its)。
2. 冠詞發音偵測器：a / an 100% 取決於單字開頭「發音」。地雷：an hour, an honest man, a university。
3. 不可數名詞禁忌：抽象概念/整體總稱（furniture, information, advice, equipment）。嚴禁加 s、嚴禁加 a/an、永遠視為單數。

## 模組二：路由座標與邏輯閘 (Routing & Logic Gates)
1. 空間與時間座標 (In/On/At)：
   - In (容器內/大範圍)：in 2026, in Taiwan.
   - On (接觸表面/特定日)：on Monday, on the desk.
   - At (精確節點/刻度)：at 10:30, at the station.
2. 向量與指標：To (方向/目的), Of (從屬/抽取), Through (3D穿透), Across (2D跨越).
3. 邏輯閘衝突禁忌：Because (因) 與 So (果) 絕對不能同時存在；Although (雖然) 與 But (但是) 絕對不能同時存在。

## 模組三：系統引擎與狀態機 (Execution Engine) - 包含編譯三大鐵律
1. 動詞權限階層：
   - Be 動詞 (Admin 權限)：宣告狀態 (=)，啟動被動與進行式。
   - 助動詞 (外掛腳本)：do/does/did/will/can。負責處理否定與疑問。
   - 一般動詞 (應用程式)：無權限，只負責執行動作。
2. 時態疊加的三大鐵律 (Compiler Rules)：
   - 鐵律一：永遠只有「最前面」的第一個動詞負責承擔時間戳記（現在/過去）。
   - 鐵律二：遇到「助動詞外掛 (do/does/did/will/can)」，後面的主動詞強制【重置為原形 (Reset)】。
   - 鐵律三：遇到「完成式日誌外掛 (have/has/had)」，後面的動詞強制變成【過去分詞 (V.p.p.)】。
3. 五大狀態機與特例 (State Machine & Exceptions)：
   - 簡單式 (排程預設)：常態發生 (every day)。
   - 進行式 (活躍程序)：Be + V-ing (此時此刻 right now)。【特例警告】：狀態動詞 (know, show, like) 屬唯讀屬性，不可加 ing。
   - 完成式 (日誌累積)：Have/Has + V.p.p. (從過去累積到現在的次數或經驗)。
   - 完成進行式 (Uptime 持續追蹤)：Have/Has been + V-ing。觸發條件：for + 一段時間。代表程序從過去啟動一路執行且「仍在執行中」。
   - 過去完成式 (依賴排序/過去的過去)：Had + V.p.p.。觸發條件：By the time + 過去式。當有兩個過去的動作，先執行的必須強制宣告為此狀態。
   - 被動語態 (物件導向)：Be + V.p.p. (發出動作前，必須先 Check 主詞是否為無生命的被操作物)。
