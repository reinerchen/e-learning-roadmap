模組一：基礎變數宣告與資料型態 (Variables & Data Types)

    代名詞最高權限：內建變數（你我他）所有格自帶專屬格式，嚴禁附加 's (例如：hers, yours, its)。

模組二：路由座標與邏輯閘 (Routing & Logic Gates)

    空間與時間座標 (In/On/At)：

        In (容器內/大範圍)：in 2026, in Taiwan.

        On (接觸表面/特定日)：on Monday, on the desk.

        At (精確節點/刻度)：at 10:30, at the station.

    邏輯閘衝突禁忌：Because/So 互斥；Although/But 互斥。

模組三：系統引擎與狀態機 (Execution Engine) - 包含編譯三大鐵律

    動詞權限階層：

        Be 動詞 (Admin 權限)：宣告狀態 (=)，啟動被動與進行式。

        助動詞 (外掛腳本)：do/does/did/will/can。負責否定與疑問。

        一般動詞 (應用程式)：無權限，只負責執行動作。

    時態疊加的三大鐵律 (Compiler Rules)：

        鐵律一：永遠只有「最前面」的第一個動詞負責承擔時間戳記（現在/過去）。

        鐵律二：遇到「助動詞外掛 (do/does/did/will/can)」，後面的主動詞強制【重置為原形 (Reset)】。

        鐵律三：遇到「完成式日誌外掛 (have/has/had)」，後面的動詞強制變成【過去分詞 (V.p.p.)】。

    進程委託 (Causative Verbs - Patch 3.4)：

        主動委託 (Active Call)：have + [Worker] + [原形動詞] (委託執行)。

        狀態回傳 (Callback)：have + [Object] + [V.p.p.] (確認完成)。

模組四：嵌套子句 (Nested Statements) - 句中句架構

    間接問句 (Nested Queries)：將疑問句作為參數傳入。

        語序重置：進入嵌套狀態後，必須還原為直述句語序 (Subject + Verb)。

        布林開關：Yes/No 問句嵌入時，必須手動引入 if 或 whether。

    關係子句 (Pointers / 關代)：為名詞掛載屬性擴充。

        指標型別：who (人), which (物), that (通用), whose (所有格屬性)。

        權限與省略：主格指標不可省略（避免雙動詞衝突）；受格指標可省略（代碼優化）。
